# Contexto del Proyecto: ai-evals-manager
Objetivo: Prototipado rápido de herramientas de evaluación de IA.

## 1. Stack Tecnológico (Estricto)
- Frontend: React + TypeScript + Vite.
- Base de Datos: Supabase (PostgreSQL).
- Librerías clave: TailwindCSS, Lucide-React, Supabase-JS.

## 2. Protocolo de Trabajo "DB-First" (Obligatorio)
La base de datos es la fuente de la verdad.
1. Antes de cualquier código nuevo que implique lectura/escritura, primero debes proponer o verificar el esquema SQL.
2. No asumas que las tablas o columnas existen. Si no lo sabes, pregúntame o verifica el esquema.
3. El SQL debe incluir `IF NOT EXISTS` para ser idempotente.
4. Siempre debes avisar al usuario: "Ejecuta este SQL en Supabase y refresca el Schema Cache".

## 3. Reglas de Programación Defensiva (Para evitar crashes)
- En `JSON.parse()`, siempre envuelve el código en un bloque `try/catch`.
- Al renderizar objetos, usa encadenamiento opcional (`config?.provider`) o valores por defecto (`|| "N/A"`).
- Nunca asumas que los datos que vienen de la base de datos están limpios; valida siempre antes de renderizar.

## 4. Estilo y Calidad
- Prioriza la mantenibilidad: Si una lógica se repite, extrae el componente o la función.
- Si hay errores, no intentes adivinar: muestra primero el log de la consola y explica el proceso de depuración.
